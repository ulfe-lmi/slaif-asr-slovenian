# Work Orders

A work order is the contract between the strategic model and an execution agent.

Rules:

- one implementation task per work order;
- current repository state must be verified;
- goal, scope, non-goals, tests, documentation, and report format are explicit;
- implementation normally produces one branch and one pull request;
- work orders are preserved so decisions and evidence remain auditable.

Use [`TEMPLATE.md`](TEMPLATE.md) for new work.
